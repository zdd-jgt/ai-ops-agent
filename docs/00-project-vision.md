# 项目愿景

项目名称：**智能运维大脑（AI Ops Agent）**。

## 背景

企业运维数据通常分散在指标、日志、链路追踪、云平台、Kubernetes、工单和人员经验中。传统告警能够发现单点异常，但难以自动回答：

- 多条告警是否属于同一故障？
- 受影响的业务和依赖范围是什么？
- 故障前发生过哪些发布或配置变更？
- 哪个根因候选最有证据支持？
- 下一步应该执行什么只读检查或处置步骤？
- 事件是否恢复、是否复发、经验是否沉淀？

智能运维大脑的价值是组织这些事实并缩短判断路径，而不是脱离监控数据进行自由推测。

## 产品目标

建设一个可以独立部署在 AWS 或客户本地环境中的智慧运维大脑，逐步提供：

1. 异常分析：识别异常模式并聚合同源信号。
2. 根因定位：基于拓扑、变更和历史事件生成根因候选。
3. 性能优化：发现瓶颈并给出有依据的优化建议。
4. 安全风险识别：汇总安全发现并分析影响范围和优先级。
5. 事件与工单：生成事件、通知负责人、记录审批和处理进度。
6. 安全执行：在权限、审批、审计和回滚约束下调用运维工具。

## 目标用户

- 运维与 SRE 团队。
- 应用开发团队。
- 安全运营团队。
- 架构师和技术负责人。
- 需要本地数据驻留或完全断网部署的企业客户。

## 核心价值

- 降低告警噪声。
- 缩短平均发现时间和平均恢复时间。
- 让诊断结论带有证据链，而不是只给自然语言答案。
- 将个人运维经验沉淀为可复用知识和标准流程。
- 兼容云原生和传统基础设施。
- 在私有化场景中保持数据主权和离线可运行能力。

## 未来效果指标

具体目标值待真实数据基线建立后确定，建议跟踪：

- 告警压缩率。
- 事件聚合准确率。
- 异常误报率和漏报率。
- 首次根因候选命中率。
- 人工采纳建议比例。
- 平均发现时间 MTTD。
- 平均确认时间 MTTA。
- 平均恢复时间 MTTR。
- 自动生成事件的完整率。
- 自动执行成功率、回滚率和人工中止率。
