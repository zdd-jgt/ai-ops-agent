# 非功能要求

## 文档状态

以下数值是第一期推荐基线，不是已经对客户承诺的 SLA。获得真实节点数、遥测量、并发和模型后必须重新压测和确认。

## 部署等级

| 等级 | 用途 | 可用性定位 |
|---|---|---|
| Lite Dev | 当前开发电脑、模拟数据和极小数据量 | 不提供 SLA |
| Offline POC | 单节点、本地模型、功能验证 | 允许维护停机，不承诺高可用 |
| Small Production | 多平台节点、独立推理和存储 | 推荐目标 99.5%，待客户确认 |

## 可用性与恢复

第一期推荐目标：

- 事件接入失败不能静默丢失，必须重试或进入可查询的失败队列。
- 同一事件重复投递不会创建重复 Incident。
- Agent Runtime 或 MCP Server 重启后，未完成 Workflow 可以恢复或明确失败。
- Small Production 推荐 RPO 不超过 15 分钟、RTO 不超过 4 小时，最终按客户等级调整。
- 备份必须定期执行恢复演练，不能只验证备份任务成功。
- 监控后端故障时，事件中心仍可查看既有 Incident 和审计记录。

## 性能推荐基线

不包含第三方后端本身的慢查询时间时：

| 操作 | 第一阶段推荐目标 |
|---|---|
| 告警接入响应 | P95 小于 500ms |
| 告警到 Incident 创建 | 正常负载下 30 秒内 |
| Incident 普通查询 | P95 小于 1 秒 |
| 指标 MCP 查询 | P95 小于 5 秒 |
| 日志 MCP 查询 | P95 小于 10 秒 |
| 单次诊断首个进度事件 | 5 秒内 |
| 单次完整诊断 | 通常 90 秒内，硬超时 5 分钟 |

超过目标时必须返回进度、取消能力或明确超时，不能让客户端无限等待。

## Agent 与 MCP 资源限制

第一期推荐：

- 单次诊断最多 20 次 MCP 工具调用。
- 相同诊断步骤最多重试 2 次。
- Agent 自主循环最多 3 轮。
- 单个工具默认超时 15 秒，日志类查询可单独配置。
- 单个 MCP 结构化响应建议不超过 256KB。
- 原始日志、Trace 和大对象不写入 Workflow Snapshot，只保存引用。
- 每个租户和环境配置独立并发与查询预算。
- 用户可以取消尚未进入不可中断阶段的诊断任务。

这些数值必须配置化，并在性能测试后调整。

## 数据保留

| 数据 | 推荐策略 |
|---|---|
| 原始指标、日志、Trace | 按客户容量和合规要求配置，默认不写死 |
| Incident 与状态时间线 | 长期保存，建议不少于 1 年，待确认 |
| Evidence 摘要和引用 | 与 Incident 生命周期一致 |
| Agent Workflow 状态 | 完成后按策略归档，失败和审批中状态优先保留 |
| 模型与 MCP Trace | 默认短期保存并脱敏，具体期限待确认 |
| 审计记录 | 建议不少于 180 天，合规客户单独确认 |

数据删除必须覆盖主存储、索引、缓存、备份和向量库，并保留合规所需的删除审计。

## 扩展性

- 接入、事件聚合、Agent Runtime、MCP Server、模型推理和存储可以独立扩容。
- 所有跨组件消息携带 `tenant_id`、`deployment_id`、`event_id` 和 Trace ID。
- 高基数标签、超大日志和超长 Trace 在入口受限。
- 第一阶段不为了假设规模提前拆成大量微服务。
- 从单体到分布式部署必须保留相同领域契约和幂等语义。

## 安全与隔离

- 租户、环境和角色权限在服务端强制执行。
- 默认最小权限和默认拒绝。
- 所有公网出口均显式配置、可审计和可禁用。
- 敏感字段过滤失败时拒绝云模型调用。
- 审计记录包含主体、动作、目标、结果、时间和关联 ID。
- Agent Trace 不得默认保存完整密钥、Prompt 原文或未脱敏工具结果。

## Air-Gapped 要求

- 本地模型和 Agent Runtime 启动前必须完成主机、GPU/驱动、磁盘、运行时及模型权重预检。
- 任何必需项低于模型规格清单要求或无法确认时失败关闭，不允许跳过、自动降级或回退公网模型。
- 断网状态下完成安装、启动、诊断、审批、备份和恢复。
- DNS、NTP、镜像、模型、许可证和前端静态资源均有本地方案。
- 不因无法连接公网而产生持续高频重试。
- 离线升级包可校验、可预检查、可回滚。
- 禁用所有 Cloud Exporter、在线分析、动态注册表和远程模型发现。
- 断网验证必须使用网络层阻断，而不只是配置上声称未调用公网。

## 兼容与升级

- API、事件和 MCP Schema 必须带版本。
- 数据库迁移提供向前执行、失败停止和恢复说明。
- 至少支持当前版本向前一个受支持版本升级。
- 离线包记录 Kubernetes、操作系统、GPU 驱动、模型和依赖兼容矩阵。
- 重大协议或数据模型变化需要迁移验证和回滚演练。

## 产品自身可观测性

至少监控：

- 事件接入成功率、延迟和失败队列。
- 事件聚合数量、重复率和规则耗时。
- Agent Run 成功、失败、取消、超时和暂停数量。
- MCP 工具延迟、错误率、拒绝率和响应大小。
- 模型延迟、Token、结构化输出失败和降级次数。
- Workflow Snapshot 数量和长期暂停任务。
- PostgreSQL、对象存储和遥测后端容量。
- 离线包版本、证书、许可证和模型有效性。

## 容量确认输入

正式定规格前必须获得：

- 集群、节点、服务、Pod、虚拟机和设备数量。
- 每秒指标样本、标签基数、每日日志和 Trace 写入量。
- 数据保留、压缩、副本和备份策略。
- 同时活动 Incident、用户和 Agent Run 数量。
- 模型、量化、上下文长度、并发和延迟目标。
- 单租户还是多租户，以及合规和隔离要求。
